Die Magie der dynamischen Werte 🪄

{{$json.name}} + {{$now}} = Automatisierung 🚀

Mach deine Workflows intelligent und dynamisch!

Was sind Expressions? 🤔

Expressions nutzen JavaScript in speziellen Klammern {{ }}, um deine Workflows dynamisch zu machen:

  • Dynamische Werte statt statischer Text
  • Zugriff auf Daten aus vorherigen Nodes mit der N8N-eigenen Syntax
  • JavaScript-Funktionen für Berechnungen und Transformationen
  • Bedingungen und Logik für intelligente Entscheidungen

📝 Einfaches Beispiel:

// Statischer Text: Hallo Max // Mit N8N Expression: Hallo {{$json.name}} // N8N nutzt eine spezielle Syntax, die JavaScript innerhalb // der {{ }} Klammern erlaubt!

Ergebnis: Hallo Anna (wenn name = "Anna")

Die N8N Expression-Syntax 📝

N8N nutzt eine eigene Expression-Sprache mit JavaScript-Support

{{ }}

Einfache Expression

{{$json.email}}
🔗

Text + Expression

Email: {{$json.email}}
🎯

Mehrere Expressions

{{$json.firstName}} {{$json.lastName}}

💡 Gut zu wissen:

N8N hat eine eigene Expression-Sprache entwickelt, die dir erlaubt, mit $json auf Daten zuzugreifen und gleichzeitig alle JavaScript-Funktionen zu nutzen!

Der magische $json 🎯

$json ist dein wichtigster Helfer - er enthält die Daten vom vorherigen Node!

📥 Input vom vorherigen Node:

{ "name": "Lisa Müller", "email": "lisa@example.com", "age": 28, "address": { "city": "Berlin", "zip": "10115" } }

✨ So greifst du darauf zu:

{{$json.name}}
Lisa Müller
{{$json.address.city}}
Berlin

Text manipulieren 📝

Mit Expressions kannst du Texte auf vielfältige Weise bearbeiten:

Häufige String-Operationen:

// Groß-/Kleinschreibung {{$json.name.toUpperCase()}} // LISA MÜLLER {{$json.name.toLowerCase()}} // lisa müller // Text ersetzen {{$json.email.replace('@', ' [at] ')}} // Text aufteilen {{$json.fullName.split(' ')[0]}} // Vorname // Text kürzen {{$json.description.substring(0, 100)}}... // Leerzeichen entfernen {{$json.input.trim()}} // Text verbinden {{$json.firstName + ' ' + $json.lastName}} // Text Länge prüfen {{$json.text.length}} // Enthält Prüfung {{$json.email.includes('@gmail.com')}}

💡 Tipp: Du kannst alle JavaScript String-Methoden verwenden!

Rechnen mit Expressions 🧮

Führe Berechnungen direkt in deinem Workflow durch:

Grundrechenarten

{{$json.price * 1.19}} // Mit MwSt {{$json.total - $json.discount}} {{$json.quantity * $json.price}} {{$json.sum / $json.count}}

Math-Funktionen

{{Math.round($json.value)}} // Runden {{Math.floor($json.price)}} // Abrunden {{Math.ceil($json.price)}} // Aufrunden {{Math.abs($json.number)}} // Betrag

🎯 Praktisches Beispiel: Preisberechnung

// Einfache Berechnung mit MwSt {{($json.basePrice * $json.quantity - $json.discount) * 1.19}} // Mit Rundung auf 2 Nachkommastellen {{Math.round((($json.basePrice * $json.quantity - $json.discount) * 1.19) * 100) / 100}} // Oder mit toFixed für Währung {{(($json.basePrice * $json.quantity - $json.discount) * 1.19).toFixed(2)}}

Berechnet: (Grundpreis × Menge - Rabatt) × MwSt

If-Else in Expressions 🔀

Nutze den Ternary Operator für Bedingungen:

Syntax: Bedingung ? Wahr : Falsch

{{$json.age >= 18 ? 'Volljährig' : 'Minderjährig'}}

Praktische Beispiele:

// Status-Prüfung {{$json.status === 'active' ? '✅ Aktiv' : '❌ Inaktiv'}} // Preiskategorie {{$json.price > 100 ? 'Premium' : 'Standard'}} // Begrüßung nach Tageszeit {{new Date().getHours() < 12 ? 'Guten Morgen' : 'Guten Tag'}} // Verschachtelte Bedingungen {{$json.score >= 90 ? 'A' : $json.score >= 80 ? 'B' : 'C'}} // Mit Default-Wert bei fehlendem Feld {{$json.name ? $json.name : 'Unbekannt'}} // Oder kürzer mit Nullish Coalescing {{$json.name ?? 'Unbekannt'}}

Zeit & Datum meistern ⏰

N8N macht die Arbeit mit Datum und Zeit super einfach:

Aktuelles Datum

{{$now}} {{$today}} {{new Date()}}

Formatierung

{{$now.format('DD.MM.YYYY')}} {{$now.format('HH:mm')}} {{$today.format('dddd')}}

Berechnungen

{{$now.plus(7, 'days')}} {{$now.minus(1, 'month')}} {{$now.startOf('week')}} {{$now.endOf('month')}}

📅 Format-Optionen:

DD = Tag (01-31) | MM = Monat (01-12) | YYYY = Jahr
HH = Stunde (00-23) | mm = Minute | ss = Sekunde
dddd = Wochentag | MMMM = Monatsname

Arrays & Objekte 📦

Arbeite mit Listen und komplexen Datenstrukturen:

Array-Operationen:

// Anzahl der Elemente {{$json.items.length}} // Erstes/Letztes Element {{$json.items[0]}} {{$json.items[$json.items.length - 1]}} // Array zu String {{$json.tags.join(', ')}} // Prüfen ob Element enthalten {{$json.tags.includes('wichtig')}} // Map - Alle Preise mit MwSt {{$json.products.map(p => p.price * 1.19)}} // Filter - Nur Preise über 50 {{$json.products.filter(p => p.price > 50)}} // Summe berechnen {{$json.prices.reduce((sum, price) => sum + price, 0)}} // Find - Erstes Element mit Bedingung {{$json.users.find(u => u.email === 'admin@example.com')}} // Some - Prüft ob mindestens ein Element die Bedingung erfüllt {{$json.orders.some(o => o.status === 'pending')}}

💡 Pro-Tipp: Nutze .map(), .filter() und .reduce() für komplexe Transformationen!

Zwischen Nodes springen 🏃‍♂️

Greife auf Daten von jedem beliebigen Node in deinem Workflow zu:

Die $node Syntax:

{{$node["Node Name"].json.fieldName}}

Praktische Beispiele:

// Daten vom "HTTP Request" Node {{$node["HTTP Request"].json.status}} // Alle Items von einem Node {{$node["Google Sheets"].all()}} // Erstes Item eines Nodes {{$node["Database"].first().json.id}} // Letztes Item {{$node["API Call"].last().json}} // Bestimmtes Item (z.B. das dritte) {{$node["Split In Batches"].item(2).json}} // Zugriff mit Variable als Node-Name {{$node[$json.sourceNode].json}} // Prüfen ob Node Daten hat {{$node["Filter"].all().length > 0}}

⚠️ Wichtig: Der Node-Name muss exakt übereinstimmen (inkl. Leerzeichen)!
